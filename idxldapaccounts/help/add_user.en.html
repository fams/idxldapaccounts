Create user screen

This screen is a simple form where you may enter the essential parameters for a user creation

Parameters

  • Organization:

    This is the Organization Unit under which the user will be created, it may be selected within all OU that reside under the "LDAP users search base" configured OU. By default, the selected OU is the one that was active in the users listing screen when you clicked on the "Add user button".

  • User id:(uid)

    A word without space characters, usually built by concatenation of the first letter of the user's first name and the user's last name (nut this can be what ever you want).

    This parameter must be unique in the LDAP.

    This parameter is used as what is called the "login" by various accounts (Unix, Samba, Windows NT ...).

  • Full name:(cn)

    The user's full name.

  • Last name:(sn)

    The user's last name.

  • Description:(description)

    A brief description for this user.

  • Password:(userPassword)

    The password that will be used by this user to access his various accounts (Unix, Windows NT ...)

  • Retype password:

    Enter the password again, both values will be compared to be sure you really typed what you wanted to, which is not so easy to verify yourself as passwords are not displayed as you type them.

  • Uid number:(uidNumber)

    A unique (in the LDAP) number greater than the configured "minimum uid number" parameter, a valid one will be proposed to you when you first access the form and if you change it to a non valid one, another valid one will be proposed to you again.

  • Primary group:(gidNumber)

    The group the user belongs to in first place, you may directly type its name (cn) or select it by clicking on the "Select" button. This button opens a groups listing with their descriptions and a search form to search the group you want.

User creation

When you think the user's parameters are corrects, click on the "Create user" button at the bottom of the screen to effectively create the user.

If some parameters are incorrect, an error screen is displayed and you may correct you errors by redisplaying the creation form with the "Back" button of you browser.

If there was no error, an entry is created in the LDAP for this user and his home directory is created on the local (to Webmin) system. If this directory, as other parameters with default values doesn't suit your needs, you may modify it within the edition screen for this user.

If user creation was successfull, you access the edition screen for this user which displays a message confirming the success of the operation.
