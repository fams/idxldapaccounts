Groups screen

The main groups screen presents you with a search form, add and delete buttons and groups listing.

Display options

By default, groups are ordered by cn, (called "name" through the interface), but you can change this by clicking on column headers that allow it (like the "Group number" header).

When no search parameter is given (that is when first arriving to this screen or when clicking on the "Show complete list button"), the maximum number of groups is displayed, this number is specified in the module configuration as the "maximum number of results displayed by a LDAP search" parameter.

If the number of groups to be displayd exceeds the "maximum number of list items displayed per page" configuration parameter, results are displayed on multiple pages with a menu allowing direct access to each page.

Search

The search form allow to filter groups based on the attribute specified in the "search by" select box.

On non numeric attributes, you can use LDAP wildcards in the search text box.

Search is always performed under the LDAP Organizational Unit specified by the "LDAP groups search base" configuration parameter, but you can restrict it to a sub OU by selecting it in the OU tree displayed at the left of the search form.

When a search result is displayed, you may return to the full groups listing by clicking on the "Show complete list" button that will then appear under the search form.

Adding groups

Under the search form, you will find a "Add group" button which leads you to the group creation form. A specific help screen is accessible from this screen.

Editing groups

The first column of the groups listing displays groups cn as links, when clicking on such a link, you will go to the edition screen for the corresponding group where you may modify his parameters. A specific help screen is accessible from this screen.

Deleting groups

The last column of the groups listing provides a "delete" button for each group, when clicking on it, you will go to a confirmation page for completly removing the corresponding group from LDAP.
